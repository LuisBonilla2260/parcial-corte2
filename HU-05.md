# Historia de Usuario - HU-05

## Backend con CRUD Completo y Documentación Swagger

### **Descripción:**
Como desarrollador, quiero un backend que implemente un CRUD completo para gestionar clientes, mesas y reservas, y que esté documentado con Swagger, de manera que pueda interactuar fácilmente con la API y realizar pruebas.

---

## **Criterios de Aceptación:**

1. **Funcionalidad del Backend:**
   - El backend debe implementar las operaciones CRUD (Crear, Leer, Actualizar, Eliminar) para las entidades:
     - Clientes
     - Mesas
     - Reservas

2. **Documentación Swagger:**
   - El backend debe incluir una documentación Swagger accesible desde `/swagger-ui.html`.
   - La documentación debe describir todos los endpoints disponibles, sus parámetros y respuestas.
   -La documentacion podrá ser accesible al momento de la ejecución del proyecto, ingresando mediante la URL http://localhost:9000/swagger-ui.html

3. **Validación:**
   - El backend debe validar los datos de entrada para cada operación.
   - Si los datos son inválidos, debe devolver un mensaje de error claro.

4. **Base de Datos:**
   - El backend debe conectarse a una base de datos PostgreSQL.
   - Las tablas deben estar correctamente diseñadas para soportar las operaciones CRUD.

---

## **Definición de Hecho:**
- El backend está implementado y probado.
- Todos los endpoints están documentados en Swagger.
- Las operaciones CRUD funcionan correctamente y están validadas.
- La base de datos está configurada y contiene las tablas necesarias.

---

## **Diseño del Backend:**

### **Estructura de las Tablas:**

#### **Tabla: Clientes**
```sql
CREATE TABLE clientes (
  id SERIAL PRIMARY KEY,
  nombre VARCHAR(100) NOT NULL,
  telefono VARCHAR(15) NOT NULL,
  email VARCHAR(100),
  status BOOLEAN DEFAULT TRUE
);
```

#### **Tabla: Mesas**
```sql
CREATE TABLE mesas (
  id SERIAL PRIMARY KEY,
  numero INT NOT NULL,
  capacidad INT NOT NULL,
  status BOOLEAN DEFAULT TRUE
);
```

#### **Tabla: Reservas**
```sql
CREATE TABLE reservas (
  id SERIAL PRIMARY KEY,
  mesa_id INT REFERENCES mesas(id),
  cliente_id INT REFERENCES clientes(id),
  fecha_reserva TIMESTAMP NOT NULL,
  duracion INT NOT NULL,
  status BOOLEAN DEFAULT TRUE
);
```

---

### **Endpoints CRUD:**

#### **Clientes**
- `POST /clientes`: Crear un cliente.
- `GET /clientes`: Obtener todos los clientes.
- `GET /clientes/{id}`: Obtener un cliente por ID.
- `PUT /clientes/{id}`: Actualizar un cliente.
- `DELETE /clientes/{id}`: Eliminar un cliente.

#### **Mesas**
- `POST /mesas`: Crear una mesa.
- `GET /mesas`: Obtener todas las mesas.
- `GET /mesas/{id}`: Obtener una mesa por ID.
- `PUT /mesas/{id}`: Actualizar una mesa.
- `DELETE /mesas/{id}`: Eliminar una mesa.

#### **Reservas**
- `POST /reservas`: Crear una reserva.
- `GET /reservas`: Obtener todas las reservas.
- `GET /reservas/{id}`: Obtener una reserva por ID.
- `PUT /reservas/{id}`: Actualizar una reserva.
- `DELETE /reservas/{id}`: Eliminar una reserva.

### **Configuración Swagger:**
```java
// filepath: c:\Users\Usuario\Desktop\FRONTS\BackPM\src\main\java\com\example\config\SwaggerConfig.java
package com.example.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import springfox.documentation.builders.PathSelectors;
import springfox.documentation.builders.RequestHandlerSelectors;
import springfox.documentation.spi.DocumentationType;
import springfox.documentation.spring.web.plugins.Docket;

@Configuration
public class SwaggerConfig {
    @Bean
    public Docket api() {
        return new Docket(DocumentationType.OAS_30)
                .select()
                .apis(RequestHandlerSelectors.basePackage("com.example"))
                .paths(PathSelectors.any())
                .build();
    }
}
```
