# Historia de Usuario - HU-03

## Seleccionar Mesa para Reservas

### **Descripción:**
Como usuario del sistema, quiero poder seleccionar una mesa específica para realizar una reserva, de manera que pueda elegir la mesa que mejor se adapte a mis necesidades.

---

## **Criterios de Aceptación:**

1. **Visualización del Componente:**
   - El sistema debe mostrar un campo desplegable para seleccionar una mesa.
   - El campo debe estar etiquetado como "Mesa".

2. **Interacción:**
   - El usuario debe poder abrir el desplegable y seleccionar una mesa de la lista.
   - La lista debe mostrar las mesas disponibles con identificadores claros.

3. **Validación:**
   - El sistema debe registrar correctamente la mesa seleccionada.
   - Si no se selecciona una mesa, el valor debe permanecer vacío.

4. **Compatibilidad:**
   - El componente debe ser funcional en dispositivos móviles y de escritorio.

---

## **Definición de Hecho:**
- El componente de selección de mesa está implementado y probado.
- La funcionalidad es accesible desde la página de gestión de reservas.
- La mesa seleccionada se almacena correctamente en el estado del componente.

---

## **Diseño del Componente:**

### **Archivo:** `SeleccionMesa.tsx`
```tsx
import React from 'react';
import { IonItem, IonLabel, IonSelect, IonSelectOption } from '@ionic/react';

const SeleccionMesa: React.FC = () => {
  return (
    <IonItem>
      <IonLabel>Mesa</IonLabel>
      <IonSelect placeholder="Seleccione una mesa">
        <IonSelectOption value="mesa1">Mesa 1</IonSelectOption>
        <IonSelectOption value="mesa2">Mesa 2</IonSelectOption>
        <IonSelectOption value="mesa3">Mesa 3</IonSelectOption>
      </IonSelect>
    </IonItem>
  );
};

export default SeleccionMesa;
```

## **Pruebas:**

1. **Prueba de Selección:**
   - Seleccionar una mesa del desplegable y verificar que el valor se almacene correctamente.
